Фоновый судья Haiku · async

После ответа отдельным фоновым вызовом Haiku оценивает полезность (usefulness_score) и пишет вердикт в лог. Пользователя не задерживает.

Сквозная инфраструктура · работает на каждом шаге

⚙️Claude CLI (Pro)

  • Все вызовы — subprocess claude --print --output-format text --no-session-persistence --disallowed-tools --model <id>
  • Каждый вызов — свой tempfile.TemporaryDirectory как cwd
  • Cross-process слот-пул на fcntl.flock, N=4 (/tmp/teplodar_claude_slots) — общий cap для бота, админки, скриптов
  • Токен — чистый потребитель, расшарен с ArkadiyJarvis (симлинк data/)

📚База знаний

  • SQLite base/teplodar.db
  • numpy-индекс embeddings_e5-base-v1.npy + metadata_e5-base-v1.pkl
  • 2961 чанк — 2191 товар / 770 PDF+HTML (после dedup 0.92)
  • E5 multilingual-base, 768d, на CPU; прогрет в память

♻️Общий генератор

  • Один AnswerGenerator-синглтон на всю RAG-логику
  • Бот строит свой в main_bot.py
  • Веб-чат и eval делят get_eval_generator()
  • Прогревается в lifespan админки → E5 постоянно в RAM
⚡
«Стриминг» — фейковый. CLI в режиме --print отдаёт ответ одним блоком в конце, поэтому в UX летят только события фаз (intent → retrieval → answer), а не токены. Настоящие дельты появятся только при переходе на API/OpenRouter.
